## 2. 주요 Q&A (FAQ)
오늘 대화 중 논의되었던 주요 질문과 답변을 기록합니다.

### Q: `docker-compose up --build`는 웹서버 없이도 가능한가요?
**A: 네, 가능합니다.**
컨테이너 내부에 Node.js 런타임이 포함되어 있어, 별도의 Apache/Nginx 설치 없이도 스스로 웹 서버 역할을 수행합니다.

### Q: 리눅스 서버 부팅 시 자동 실행하려면?
**A: 두 단계가 필요합니다.**
1.  **Docker 데몬 활성화**: `sudo systemctl enable docker` (운영체제 레벨)
2.  **컨테이너 재시작 정책**: `docker-compose.yml`에 `restart: always` 추가 (Docker 레벨)

### Q: 현재 터미널 실행 vs Docker 실행 차이점?
**A:**
*   **현재(로컬)**: 내 컴퓨터의 Node.js를 직접 사용. 개발 편의성 높음.
*   **Docker**: 격리된 컨테이너 환경 사용. 환경 일관성 및 배포 편의성 높음.

### Q: 패키지 정보는 어디에 있나?
**A: `package.json` 파일**
Backend와 Frontend 폴더 각각에 있는 `package.json`이 설치할 패키지 목록(영수증) 역할을 합니다. Docker는 이를 보고 자동으로 설치합니다.

### Q: `sudo systemctl` 명령어 실행 위치?
**A: 리모트 리눅스 서버 터미널**
내 컴퓨터(윈도우)가 아니라, 배포할 리눅스 서버에 SSH로 접속해서 입력해야 합니다.
