## 2. 프로젝트 개요 (문서 기반 분석)
`USER_CASE.md`와 `TEST_CASE.md` 내용을 바탕으로 유추한 프로젝트의 핵심 내용은 다음과 같습니다.

### 2.1 제품 비전
**"개발자 프로파일 블로그 & 앱 확장 플랫폼"**
개발자의 포트폴리오와 기술 블로그를 웹으로 제공하고, 이를 Flutter WebView를 통해 모바일 앱으로도 확장하여 배포하는 하이브리드 플랫폼입니다.

### 2.2 핵심 기술 스택
- **Web**: Next.js (SSR, SEO 최적화, MDX 블로그 예상)
- **Backend**: Node.js (API 서버 - 프로젝트/블로그 데이터 제공, Contact 폼 처리)
- **Mobile**: Flutter (WebView 기반 - Android/iOS 지원, 네이티브 기능 일부 연동 가능성)

### 2.3 주요 사용자 (Actors)
1. **Visitor (방문자)**: 일반적인 정보 탐색
2. **Recruiter (채용 담당자)**: 기술 역량 및 이력 검증
3. **Collaborator (협업 제안자)**: 프로젝트 의뢰 및 컨택
4. **Owner (관리자)**: MDX 콘텐츠 작성 및 배포 관리

### 2.4 핵심 기능
1.  **메인 홈**: Hero 섹션, 기술 스택 시각화, 타임라인(경력/이력).
2.  **프로젝트**: 리스트/상세 보기, 태그 필터링, 관련 링크(GitHub/Demo).
3.  **블로그**: 기술 포스팅 목록/상세, 목차(TOC), 코드 하이라이팅.
4.  **검색(Search)**: 블로그 및 프로젝트 통합 검색.
5.  **연락(Contact)**: 유효성 검사 및 Rate Limit이 적용된 문의 폼.
6.  **앱 기능**: WebView 기반 탐색, 시스템 뒤로가기 연동, 외부 링크 처리, Safe Area 대응.

## 3. 제언 및 다음 단계
현재 구현체는 없고 테스트/사용자 케이스만 정의된 상태입니다. 다음 단계로 아래 작업을 추천합니다.

2.  **프로젝트 스캐폴딩(Scaffolding)**:
    - `frontend` (Next.js) 생성
    - `backend` (Node.js) 생성
